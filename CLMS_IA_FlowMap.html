Data Flow — Clerk Actions

enrichBook(id, data) → updates booksMock enrichment
↓ Barrister FilterPillBar derives new filter options (IX-5)

approveLoan(id) → status: pending → active
↓ Barrister sees active loan

denyLoan(id, reason) → status: denied
sendReminder(id) → toast, no data change

addBook() / importCSV()
↓ Adds to booksMock, appears in search

inviteMember(email, role)
↓ Adds to membersMock

IX-5 Value Loop

Clerk enriches book (Library → BookDetailPanel)
↓ booksService.enrichBook() mutates booksMock
↓ Barrister opens Search
↓ FilterPillBar re-derives options from booksMock
↓ New filter option appears (e.g., "Criminal")
↓ Barrister can now filter by that subject

This is the core product loop that connects both roles.

Cross-Role Flows

How Barrister and Clerk actions connect through shared mock data layer.

IX-2 Loan Lifecycle

1. Barrister Requests

  • Search page → "Request Loan"
  • requestLoan(bookId, name)
  • Creates loan: status = pending
  • Toast: "Loan requested, Clerk notified"
  • Button: "Request Loan" → "Requested"
  • Badge: Available → Requested
→

2. Clerk Decides

  • Loans page → Pending tab
  • Approve → status = active
  • Sets dueDate (14 days from now)
  • Toast: "Approved, Borrower notified"
  • Deny → LoanActionModal
  • Reason input → status = denied
→

3. Barrister Uses

  • Loans page → Active tab
  • Shows due date, location, status
  • Extend → +7 days
  • Return → status = returned
→

4. Clerk Monitors

  • Loans page → Overdue tab
  • Send Reminder → one-click toast
  • Dashboard: overdue count in snapshot
  • Bell icon: pending + overdue count
Authority List Lifecycle

1. Search

  • Barrister or Clerk searches
  • Results: books + JADE (cases, legislation)
  • Filters powered by clerk enrichment
→

2. Add to List

  • "+ Authority List" button
  • addItem(listId, item)
  • Auto-adds to first list
  • Toast: "Added to [list name]"
  • Bottom drawer auto-expands
→

3. Build Workspace

  • Authorities page
  • Items grouped by type:
Cases Legislation Books
  • Sidebar: summary + actions
→

4. Export AGLC

  • exportAGLC(list)
  • Opens HTML in new tab
  • Formatted: TABLE OF AUTHORITIES
  • Sections: A. Cases, B. Legislation, C. Books
  • Browser print/save as PDF
Court-ready AGLC4 format
IX-5 Enrichment Value Loop

1. Clerk Imports

  • Library page: Add Book / Import CSV
  • Books enter catalogue as ISBN Only
  • No enrichment data yet
  • Barrister filters: disabled/empty
→

2. Clerk Enriches

  • Library → BookDetailPanel
  • Fills: Subject, Jurisdiction, Resource Type, Tags, Notes
  • enrichBook(id, data)
  • Badge: ISBN Only → Enriched
  • Dashboard: enrichment % goes up
→

3. Barrister Searches

  • Search page renders FilterPillBar
  • FilterPillBar reads booksMock on every render
  • Derives filter options from enrichment data
  • New subjects/jurisdictions appear automatically
  • No cache, no static list
→

4. Better Research

  • More filters = more precise search
  • Better results in authority lists
  • Higher quality court output
  • Motivates clerk to enrich more
Virtuous cycle

Shared Mock Data Layer (Service Pattern)

loansService.js

getLoans() — read all
requestLoan() — barrister creates
approveLoan() — clerk approves
denyLoan() — clerk denies
returnBook() — barrister returns
renewLoan() — barrister extends
sendReminder() — clerk reminds

booksService.js

getBooks() — read all
enrichBook() — clerk enriches
addBook() — clerk adds single
importBooks() — clerk bulk import

authorityListsService.js

getLists() — read all
addItem() — add to list
createList() — new list

Other Services

searchAll(q) — unified search
exportAGLC(list) — HTML export
getMembers() — team data
inviteMember() — add member

Toast Notification System

Every mutation triggers addToast({ message, type })

Types:
success "Loan requested", "Book enriched", "Approved"
info "Switched to barrister view", "Open in Library"
warning Phase 3+
error Real API only

Auto-dismiss: 3 seconds. Stack: top-right fixed.

Bell Notification Badge

AppShell fetches getLoans() on mount
Counts: status === 'pending' || status === 'overdue'
Shows red pulsing dot when count > 0

Both Barrister and Clerk see this badge.
Clicking bell → navigates to /app/loans
